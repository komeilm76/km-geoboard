# Page 06-05 — Export to PDF Meta

## Summary
Implements `exportToPdfMeta` — produces a `PdfMeta` object that a consumer can
pass directly to a PDF library (jsPDF, PDFKit, pdfmake). The function converts
artboard dimensions to PDF point units and generates an SVG string scaled to
the page size.

## Target
`packages/km-exports/src/exportToPdfMeta.ts` exports `exportToPdfMeta`.
Tests verify page-size conversion, orientation handling, and SVG scaling.

## Dependencies
- Page 06-02 (export-svg) — `exportToSvg` used to generate the `svgContent` field

## Inputs
- `DT-Exports.md` — `exportToPdfMeta` spec, `PdfMeta` type, page size constants.

## Outputs

|---|---|
| `packages/km-exports/src/exportToPdfMeta.ts` | `exportToPdfMeta` implementation |

## Step-by-Step Instructions

1. Create `packages/km-exports/src/exportToPdfMeta.ts`.

2. Define page size constants (in points, 1 pt = 1/72 inch):
   ```ts
   const PAGE_SIZES = {
     A4:     { width: 595.28,  height: 841.89 },
     A3:     { width: 841.89,  height: 1190.55 },
     Letter: { width: 612,     height: 792 },
     Legal:  { width: 612,     height: 1008 },
   } as const;
   ```

3. Function signature:
   ```ts
   function exportToPdfMeta(
     artboard: Artboard,
     elements: SvgElement[],
     options?: {
       pageSize?: "A4" | "A3" | "Letter" | "Legal" | { width: number; height: number };
       orientation?: "portrait" | "landscape";
       title?: string;
       author?: string;
       filter?: ExportFilter;
     }
   ): Result<PdfMeta>
   ```

4. Destructure options with defaults:
   ```ts
   const { pageSize = "A4", orientation = "portrait", title, author, filter } = options ?? {};
   ```

5. Resolve the page dimensions:
   - If `pageSize` is a string key, look it up in `PAGE_SIZES`.
   - If `pageSize` is a custom `{ width, height }` object, use directly.
   - Apply orientation: if `"landscape"` and `width < height`, swap width and height.

6. Generate the SVG content by calling `exportToSvg`:
   ```ts
   const svgResult = exportToSvg({ artboard, elements, filter, xmlDeclaration: false });
   if (!svgResult.success) return svgResult;
   ```

7. Build and return the `PdfMeta` object:
   ```ts
   return {
     success: true,
     data: {
       pageSize: { width: resolvedWidth, height: resolvedHeight },
       orientation,
       svgContent: svgResult.data,
       artboard,
       title,
       author,
     },
   };
   ```

8. Write tests:
   - A4 portrait: `pageSize.width ≈ 595`, `pageSize.height ≈ 842`.
   - A4 landscape: `pageSize.width ≈ 842`, `pageSize.height ≈ 595`.
   - Custom page size: `{ width: 300, height: 400 }` → same in output.
   - `svgContent` is a non-empty string starting with `"<svg"`.
   - `title` and `author` passed through when provided.
   - `"empty-export"` propagated from `exportToSvg` when filter removes all elements.
   - `artboard` is the same object reference in output.

## Acceptance Criteria

- [ ] A4 portrait dimensions correct (595.28 × 841.89 pt)
- [ ] Landscape swaps width and height correctly
- [ ] Custom `{ width, height }` page size is passed through unchanged
- [ ] `svgContent` is generated by `exportToSvg`
- [ ] `exportToSvg` errors propagated
- [ ] `title` and `author` are optional and preserved when provided
- [ ] All tests pass, build clean

## Notes
- `exportToPdfMeta` does NOT generate a PDF. It produces the data structure that
  a consumer's PDF library needs. The `svgContent` is a self-contained SVG string
  that the PDF library can embed on the page.
- The `svgContent` SVG uses artboard canvas units for its `viewBox`, and the
  `pageSize` uses PDF point units. The PDF library is responsible for scaling
  the SVG to fit the page — this library simply provides both pieces of
  information in one object.
