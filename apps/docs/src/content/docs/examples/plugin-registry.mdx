`@komeilm76/km-plugins` is a typed plugin registry with dependency resolution, conflict detection, and semver-aware lifecycle management. Use it to make any km-geoboard extension point pluggable.

## Live example

<LiveExample
  client:only="react"
  title="Plugin registry"
  code={`import { plugins } from '@komeilm76/km-geoboard';

// ── 1. Create a fresh registry ────────────────────────────────────────────
const registry = plugins.createRegistry();

// ── 2. Define and register plugins ───────────────────────────────────────
const corePlugin = {
  id: 'core',
  version: '1.0.0',
  name: 'Core utilities',
  dependencies: [],
};

const wktPlugin = {
  id: 'wkt-importer',
  version: '0.2.0',
  name: 'WKT importer',
  dependencies: [{ id: 'core', range: '>=1.0.0' }],
};

const gpxPlugin = {
  id: 'gpx-importer',
  version: '0.1.0',
  name: 'GPX importer',
  dependencies: [{ id: 'core', range: '>=1.0.0' }],
};

const r1 = plugins.registerPlugin(registry, corePlugin);
const r2 = plugins.registerPlugin(registry, wktPlugin);
const r3 = plugins.registerPlugin(registry, gpxPlugin);
console.log('Registered:', [r1, r2, r3].map(r => r.success ? r.plugin.id : r.reason).join(', '));

// ── 3. Resolve activation order ───────────────────────────────────────────
const order = plugins.resolveOrder(registry, ['wkt-importer', 'gpx-importer']);
if (order.success) {
  console.log('Activation order:', order.ids.join(' → '));
}

// ── 4. Activate plugins ───────────────────────────────────────────────────
for (const id of (order.success ? order.ids : [])) {
  const act = plugins.activatePlugin(registry, id);
  console.log('Activate ' + id + ':', act.success ? 'OK' : act.reason);
}

// ── 5. List registry state ────────────────────────────────────────────────
const all = plugins.listPlugins(registry);
all.forEach(p => console.log(' ' + (p.active ? '✓' : '○') + ' ' + p.id + '@' + p.version));

// ── 6. Deactivate ─────────────────────────────────────────────────────────
const deact = plugins.deactivatePlugin(registry, 'wkt-importer');
console.log('Deactivate wkt-importer:', deact.success ? 'OK' : deact.reason);
`}
/>

## Concepts

**Registry** — an immutable-style value created by `createRegistry()`. Every mutation function takes the registry and returns a new registry (plus a result). The original is never mutated.

**Dependency resolution** — `resolveOrder` produces a topologically sorted list. It fails (with the cycle members named) if circular dependencies are detected.

**Conflict detection** — `registerPlugin` fails with `'conflict'` if a plugin with the same `id` but an incompatible semver range is already registered.

**Lifecycle** — plugins start `inactive`. `activatePlugin` transitions them to `active` after checking their dependencies are also active.

## API summary

| Function | Returns | Notes |
|---|---|---|
| `createRegistry()` | `PluginRegistry` | Empty registry |
| `registerPlugin(registry, def)` | `RegisterResult` | Validates id, version, deps |
| `activatePlugin(registry, id)` | `ActivateResult` | Checks deps are active |
| `deactivatePlugin(registry, id)` | `DeactivateResult` | Checks no dependents are active |
| `resolveOrder(registry, ids)` | `ResolveOrderResult` | Topological sort; detects cycles |
| `listPlugins(registry)` | `PluginState[]` | All plugins with status |
| `getPlugin(registry, id)` | `PluginState \| undefined` | Single lookup |
