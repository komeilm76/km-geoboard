`@komeilm76/km-plugins` is a typed plugin registry with dependency resolution, conflict detection, and semver-aware lifecycle management. Each plugin exposes a typed API returned from its `setup` function.

## Live example

<LiveExample
  client:only="react"
  title="Plugin registry"
  code={`import { plugins } from '@komeilm76/km-geoboard';

// ── 1. Create a fresh registry ────────────────────────────────────────────
const registry = plugins.createPluginRegistry();

// ── 2. Define plugins — each has id, version, and a setup() function ─────
const corePlugin = {
  id: 'com.example.core',
  name: 'Core utilities',
  version: '1.0.0',
  setup(_deps) {
    return { greet: (name) => 'Hello, ' + name + '!' };
  },
};

const wktPlugin = {
  id: 'com.example.wkt',
  name: 'WKT importer',
  version: '0.2.0',
  // Declare that this plugin requires core >= 1.0.0
  dependencies: [{ pluginId: 'com.example.core', minVersion: '>=1.0.0' }],
  setup(deps) {
    const core = deps.get('com.example.core');
    return { parseWkt: (wkt) => core.greet('WKT: ' + wkt) };
  },
};

// ── 3. Register plugins ───────────────────────────────────────────────────
// Dependency check: wkt requires core — register core first
const r1 = registry.register(corePlugin);
const r2 = registry.register(wktPlugin);
console.log('core registered:', r1.success);
console.log('wkt registered:', r2.success);
if (!r2.success) console.log('Error:', r2.error.message);

// ── 4. Call the plugin APIs ───────────────────────────────────────────────
const coreApi = registry.getApi('com.example.core');
console.log('core.greet:', coreApi ? coreApi.greet('world') : 'not found');

const wktApi = registry.getApi('com.example.wkt');
console.log('wkt.parseWkt:', wktApi ? wktApi.parseWkt('POINT(1 2)') : 'not found');

// ── 5. List all registered plugins ────────────────────────────────────────
const all = registry.list();
all.forEach(reg =>
  console.log(' ' + (reg.status === 'active' ? '✓' : '✗') + ' ' + reg.plugin.id + '@' + reg.plugin.version)
);

// ── 6. Topological dependency sort ───────────────────────────────────────
// Pass an array of Plugin definitions to get an activation order
const order = plugins.resolveDependencyOrder([corePlugin, wktPlugin]);
if (order.success) {
  console.log('Activation order:', order.data.map(p => p.id).join(' → '));
}

// ── 7. Unregister (reverse dependency order) ─────────────────────────────
// Must unregister wkt before core (core has a dependent)
const u1 = registry.unregister('com.example.wkt');
console.log('Unregister wkt:', u1.success ? 'OK' : u1.error.code);
const u2 = registry.unregister('com.example.core');
console.log('Unregister core:', u2.success ? 'OK' : u2.error.code);
console.log('Plugins remaining:', registry.list().length);
`}
/>

## Concepts

**Plugin definition** — a plain object with `id`, `name`, `version`, optional `dependencies` / `conflicts`, and a `setup(deps)` function. `setup` receives a `Map` of resolved dependency APIs and must return the plugin's own API object.

**Registration** — `registry.register(plugin)` validates conflicts and dependencies, then calls `setup`. Returns `Result<PluginRegistration>`.

**Dependency resolution** — `resolveDependencyOrder(plugins)` produces a topologically sorted list. Fails with the cycle members named if circular dependencies are detected.

**Unregistration order** — always unregister dependents before their dependencies.

## API summary

| Symbol | Kind | Notes |
|---|---|---|
| `createPluginRegistry()` | Function | Returns a new `PluginRegistry` instance |
| `registry.register(plugin)` | Method | Validates + calls `setup`; returns `Result<PluginRegistration>` |
| `registry.unregister(id)` | Method | Calls `teardown`; fails if dependents still active |
| `registry.getApi<API>(id)` | Method | Returns the plugin's API, or `null` |
| `registry.has(id)` | Method | `true` if the plugin is registered |
| `registry.list()` | Method | All `PluginRegistration[]` in insertion order |
| `registry.reset()` | Method | Tears down all plugins in reverse order |
| `resolveDependencyOrder(plugins)` | Function | Topological sort; returns `Result<Plugin[]>` |
