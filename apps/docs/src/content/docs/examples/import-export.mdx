`@komeilm76/km-imports` turns raw strings and objects into typed internal structures. `@komeilm76/km-exports` serializes them back out. Together they form a round-trip pipeline.

## Live example

<LiveExample
  client:only="react"
  title="Import / export round-trip"
  code={`import { imports, exports as ex } from '@komeilm76/km-geoboard';

// ── 1. Import GeoJSON ─────────────────────────────────────────────────────
// importGeoJson accepts a parsed object or a JSON string
const raw = {
  type: 'FeatureCollection',
  features: [
    {
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [13.405, 52.52] },
      properties: { name: 'Berlin' },
    },
    {
      type: 'Feature',
      geometry: { type: 'LineString', coordinates: [[0,0],[1,1],[2,0]] },
      properties: { label: 'Triangle edge' },
    },
  ],
};

const result = imports.importGeoJson(raw);
console.log('importGeoJson ok:', result.success);
if (result.success) {
  console.log('Features imported:', result.data.features.length);
  if (result.warnings.length) {
    console.log('Warnings:', result.warnings.map(w => w.code).join(', '));
  }
}

// ── 2. importAuto (auto-detect format) ───────────────────────────────────
// Returns { format, result } — format is the detected type string
const auto = imports.importAuto(raw);
console.log('Detected format:', auto.format);
console.log('importAuto ok:', auto.result.success);

// ── 3. Round-trip: export back to GeoJSON string ─────────────────────────
// exportToGeoJson takes { features: GeoJsonFeature[], pretty? }
if (result.success) {
  const exported = ex.exportToGeoJson({ features: result.data.features, pretty: true });
  if (exported.success) {
    const parsed = JSON.parse(exported.data);
    console.log('Re-exported type:', parsed.type, '| features:', parsed.features.length);
    console.log('First feature geometry:', parsed.features[0].geometry.type);
  }
}
`}
/>

## Pipeline overview

```
raw string / object
       │
  importGeoJson()   importSvg()   importAuto()   importWkt()
       │
  { success, data: GeoJsonFeatureCollection, warnings }
       │
  exportToGeoJson({ features })   exportToSvg({ artboard, elements })
       │
  { success, data: string }   (serialized output)
```

`importAuto` probes the input and delegates to the appropriate importer — useful when the format is unknown at call time. Its result carries a `format` discriminant (`"geojson"`, `"svg"`, `"wkt"`, etc.).

## API summary

### km-imports

| Function | Input | Output |
|---|---|---|
| `importGeoJson(raw)` | Unknown object or string | `ImportResult<GeoJsonFeatureCollection>` |
| `importSvg(xml)` | SVG string | `ImportResult<SvgDocument>` |
| `importWkt(wkt)` | WKT string | `ImportResult<GeoJsonFeatureCollection>` |
| `importAuto(raw)` | Any | `AutoImportResult` — includes detected `format` |

### km-exports

| Function | Input | Output |
|---|---|---|
| `exportToGeoJson(options)` | `{ features, filter?, pretty? }` | `Result<string>` — GeoJSON string |
| `exportToSvg(options)` | `{ artboard, elements, ... }` | `Result<string>` — SVG string |
| `exportToPdfMeta(options)` | `{ artboard, elements, ... }` | `Result<PdfMeta>` |
