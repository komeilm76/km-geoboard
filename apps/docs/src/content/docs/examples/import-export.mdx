`@komeilm76/km-imports` turns raw strings and objects into typed internal structures. `@komeilm76/km-exports` serializes them back out. Together they form a round-trip pipeline.

## Live example

<LiveExample
  client:only="react"
  title="Import / export round-trip"
  code={`import { imports, exports } from '@komeilm76/km-geoboard';

// ── 1. Import GeoJSON ─────────────────────────────────────────────────────
const raw = {
  type: 'FeatureCollection',
  features: [
    {
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [13.405, 52.52] },
      properties: { name: 'Berlin' },
    },
    {
      type: 'Feature',
      geometry: { type: 'LineString', coordinates: [[0,0],[1,1],[2,0]] },
      properties: { label: 'Triangle edge' },
    },
  ],
};

const result = imports.importGeoJSON(raw);
console.log('importGeoJSON ok:', result.success);
if (result.success) {
  console.log('Features imported:', result.data.features.length);
}

// ── 2. importAuto (auto-detect format) ───────────────────────────────────
const autoResult = imports.importAuto(raw);
console.log('importAuto format:', autoResult.success ? autoResult.format : autoResult.reason);

// ── 3. Round-trip: export back to GeoJSON ─────────────────────────────────
if (result.success) {
  const exported = exports.exportGeoJSON(result.data);
  console.log('exportGeoJSON type:', exported.type, '| features:', exported.features.length);
}

// ── 4. Export SVG ─────────────────────────────────────────────────────────
if (result.success) {
  const svgOut = exports.exportSvg(result.data, { width: 800, height: 600 });
  console.log('exportSvg starts with <svg:', svgOut.startsWith('<svg'));
}

// ── 5. PDF metadata ───────────────────────────────────────────────────────
if (result.success) {
  const pdfMeta = exports.exportPdfMeta(result.data, { title: 'Berlin map' });
  console.log('PDF meta title:', pdfMeta.title, '| features:', pdfMeta.featureCount);
}
`}
/>

## Pipeline overview

```
raw string / object
       │
  importGeoJSON()   importSvg()   importAuto()   importOL()
       │
  FeatureCollection (typed, validated)
       │
  exportGeoJSON()   exportSvg()   exportPdfMeta()   exportOL()
       │
  serialized output
```

`importAuto` probes the input and delegates to the appropriate importer — useful when the format is unknown at call time.

## API summary

### km-imports

| Function | Input | Output |
|---|---|---|
| `importGeoJSON(raw)` | Unknown object | `ImportGeoJSONResult` |
| `importSvg(xml)` | SVG string | `ImportSvgResult` |
| `importAuto(raw)` | Any | `ImportAutoResult` (includes detected `format`) |
| `importOL(snapshot)` | OL layer snapshot | `ImportOLResult` |

### km-exports

| Function | Input | Output |
|---|---|---|
| `exportGeoJSON(fc)` | FeatureCollection | `ExportGeoJSONResult` |
| `exportSvg(fc, opts)` | FeatureCollection | `ExportSvgResult` |
| `exportPdfMeta(fc)` | FeatureCollection | `ExportPdfMetaResult` |
| `exportOL(fc)` | FeatureCollection | `ExportOLResult` |
