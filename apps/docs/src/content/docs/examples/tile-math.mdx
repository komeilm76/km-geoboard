`@komeilm76/km-map` is a pure math library for web-map computations — no map renderer required. It covers Web Mercator tile indices, coordinate projections, great-circle distance, and bounding-box helpers.

## Live example

<LiveExample
  client:only="react"
  title="Tile math"
  code={`import { map } from '@komeilm76/km-geoboard';

// ── 1. Lat/lng → tile coordinates (zoom 10) ───────────────────────────────
// Note: functions use { lat, lng } objects, not (lng, lat, zoom)
const tile = map.latLngToTile({ lat: 52.52, lng: 13.405 }, 10); // Berlin
console.log('Berlin tile z10:', tile); // { x, y, z }

// ── 2. Tile → bounding box ────────────────────────────────────────────────
const bounds = map.tileToBounds(tile); // [west, south, east, north]
console.log('Tile bounds:', bounds.map(n => n.toFixed(4)).join(', '));

// ── 3. Lat/lng → global pixel (256 px tiles) ─────────────────────────────
const px = map.latLngToPixel({ lat: 52.52, lng: 13.405 }, 10);
console.log('Pixel (global):', Math.round(px.x), Math.round(px.y));

// ── 4. Haversine distance ─────────────────────────────────────────────────
const dist = map.haversineDistance(
  { lat: 52.52, lng: 13.405 },   // Berlin
  { lat: 48.864, lng: 2.349 },   // Paris
);
console.log('Berlin → Paris (km):', (dist / 1000).toFixed(1));

// ── 5. Tiles in bounding box ──────────────────────────────────────────────
// BoundingBox is a tuple: [west, south, east, north]
const bbox = [13.3, 52.4, 13.6, 52.6];
const tiles = map.tilesForBounds(bbox, 12);
console.log('Tiles in Berlin bbox z12:', tiles.length);

// ── 6. Bounds containment ────────────────────────────────────────────────
const inside  = map.boundsContains(bbox, { lat: 52.52, lng: 13.405 });
const outside = map.boundsContains(bbox, { lat: 48.864, lng: 2.349 });
console.log('Berlin in bbox:', inside, '| Paris in bbox:', outside);
`}
/>

## Coordinate systems

All functions work in **WGS84 / EPSG:4326** degrees using `{ lat, lng }` objects. Tile XY follows the [OSM / Google Maps convention](https://wiki.openstreetmap.org/wiki/Slippy_map_tilenames) (origin top-left, Y increases downward). `BoundingBox` is a `[west, south, east, north]` tuple — the GeoJSON convention.

## API summary

| Function | Returns | Notes |
|---|---|---|
| `latLngToTile(latLng, zoom)` | `TileCoord` | Integer `{ x, y, z }` |
| `tileToBounds(tile)` | `BoundingBox` | `[west, south, east, north]` |
| `latLngToPixel(latLng, zoom)` | `PixelPoint` | Global pixel (256 px tiles) |
| `pixelToLatLng(pixel, zoom)` | `LatLng` | Inverse of above |
| `haversineDistance(from, to)` | `number` | Metres, WGS84 sphere |
| `tilesForBounds(bounds, zoom)` | `TileCoord[]` | All tiles intersecting bbox |
| `boundsContains(bounds, point)` | `boolean` | Inclusive containment |
| `latLngToMercator(latLng)` | `MercatorPoint` | Web Mercator (EPSG:3857) metres |
| `mercatorToLatLng(point)` | `LatLng` | Inverse Mercator |
